Descripción breve

Describe expresiones regulares en PowerShell.

Descripción larga

Nota

En este artículo se mostrará la sintaxis y los métodos para usar expresiones regulares en PowerShell, no se trata toda la sintaxis. Para obtener una referencia más completa, vea Lenguaje de expresiones regulares: referencia rápida.

Una expresión regular es un patrón que se usa para hacer coincidir el texto. Se puede crear con caracteres literales, operadores y otras construcciones.

En este artículo se muestra la sintaxis de expresiones regulares en PowerShell. PowerShell tiene varios operadores y cmdlets que usan expresiones regulares. Puede leer más sobre su sintaxis y uso en los vínculos siguientes.

  • Select-String
  • Operadores -match y -replace
  • -split
  • Instrucción switch con la opción -regex

De forma predeterminada, las expresiones regulares de PowerShell no tienen en cuenta las mayúsculas y minúsculas. Cada método mostrado anteriormente tiene una manera diferente de forzar la confidencialidad de mayúsculas y minúsculas.

Método Distinción entre mayúsculas y minúsculas
Select-String use -CaseSensitive switch
Instrucción switch usar la -casesensitive opción
operadores prefix con 'c' (-cmatch, -csplito -creplace)

Literales de carácter

Una expresión regular puede ser un carácter literal o una cadena. La expresión hace que el motor coincida exactamente con el texto especificado.

PowerShell
# This statement returns true because book contains the string "oo"
'book' -match 'oo'

Clase de caracteres

Aunque los literales de caracteres funcionan si conoce el patrón exacto, las clases de caracteres permiten ser menos específicos.

Grupos de caracteres

[character group] permite coincidir con cualquier número de caracteres una vez, mientras que [^character group] solo coincide con los caracteres QUE NO están en el grupo.

PowerShell
# This expression returns true if the pattern matches big, bog, or bug.
'big' -match 'b[iou]g'

Si la lista de caracteres que debe coincidir incluye el carácter de guión (-), debe estar al principio o al final de la lista para distinguirlo de una expresión de intervalo de caracteres.

Intervalos de caracteres

Un patrón también puede ser un intervalo de caracteres. Los caracteres pueden ser alfabéticos [A-Z], numéricos [0-9]o incluso basados en [ -~] ASCII (todos los caracteres imprimibles).

PowerShell
# This expression returns true if the pattern matches any 2 digit number.
42 -match '[0-9][0-9]'

Números

La \d clase de caracteres coincidirá con cualquier dígito decimal. Por el contrario, \D coincidirá con cualquier dígito no decimal.

PowerShell
# This expression returns true if it matches a server name.
# (Server-01 - Server-99).
'Server-01' -match 'Server-\d\d'

Caracteres de palabra

La \w clase de caracteres coincidirá con cualquier carácter de palabra [a-zA-Z_0-9]. Para que coincida con cualquier carácter que no es de palabra, use \W.

PowerShell
# This expression returns true.
# The pattern matches the first word character 'B'.
'Book' -match '\w'

Caracteres comodín

El punto (.) es un carácter comodín en expresiones regulares. Coincidirá con cualquier carácter excepto una nueva línea (\n).

PowerShell
# This expression returns true.
# The pattern matches any 4 characters except the newline.
'a1\ ' -match '....'

Espacio en blanco

El espacio en blanco coincide con la clase \s de caracteres. Cualquier carácter que no sea de espacio en blanco coincide con \S. También se pueden usar ' ' caracteres de espacio literal.

PowerShell
# This expression returns true.
# The pattern uses both methods to match the space.
' - ' -match '\s- '

Cuantificadores

Los cuantificadores controlan cuántas instancias de cada elemento deben estar presentes en la cadena de entrada.

Estos son algunos de los cuantificadores disponibles en PowerShell:

Cuantificador Descripción
* Cero o más veces.
+ Una o varias veces.
? Cero o una vez.
{n,m} Al menos n, pero no más de veces m .

El asterisco (*) coincide con el elemento anterior cero o más veces. El resultado es que incluso una cadena de entrada sin el elemento sería una coincidencia.

PowerShell
# This returns true for all account name strings even if the name is absent.
'ACCOUNT NAME:    Administrator' -match 'ACCOUNT NAME:\s*\w*'

El signo más (+) coincide con el elemento anterior una o varias veces.

PowerShell
# This returns true if it matches any server name.
'DC-01' -match '[A-Z]+-\d\d'

El signo de interrogación ? coincide con el elemento anterior cero o una vez. Al igual que el asterisco *, incluso coincidirá con cadenas en las que el elemento está ausente.

PowerShell
# This returns true for any server name, even server names without dashes.
'SERVER01' -match '[A-Z]+-?\d\d'

El {n, m} cuantificador se puede usar de varias maneras diferentes para permitir un control granular sobre el cuantificador. El segundo elemento m y la coma son , opcionales.

Cuantificador Descripción
{n} Coincide exactamente n con el número de veces.
{n,} Coincide al menos n con el número de veces.
{n,m} Coincidencia entre n y m número de veces.
PowerShell
# This returns true if it matches any phone number.
'111-222-3333' -match '\d{3}-\d{3}-\d{4}'

Delimitadores

Los delimitadores permiten hacer que una coincidencia se realiza correctamente o se producirá un error en función de la posición de coincidencias dentro de la cadena de entrada.

Los dos delimitadores usados habitualmente son ^ y $. El caret coincide ^ con el inicio de una cadena y $, que coincide con el final de una cadena. Los delimitadores permiten buscar coincidencias con el texto en una posición específica y, al mismo tiempo, descartar los caracteres no deseados.

PowerShell
# The pattern expects the string 'fish' to be the only thing on the line.
# This returns FALSE.
'fishing' -match '^fish$'

Nota

Al definir una expresión regular $ que contiene un delimitador, asegúrese de incluir la expresión regular mediante comillas simples (') en lugar de comillas dobles (") o PowerShell expandirá la expresión como una variable.

Al usar delimitadores en PowerShell, debe comprender la diferencia entre las opciones de expresión regular de línea única y multilínea .

  • Multilínea: el modo multilínea ^ fuerza a y $ a que coincidan con el final inicial de cada LÍNEA en lugar del principio y el final de la cadena de entrada.
  • Singleline: el modo singleline trata la cadena de entrada como singleline. Obliga al carácter a . coincidir con todos los caracteres (incluidas las líneas nuevas), en lugar de coincidir con todos los caracteres EXCEPTO con la nueva línea \n.

Para obtener más información sobre estas opciones y cómo usarlas, visite lenguaje de expresiones regulares: referencia rápida.

Caracteres de escape

La barra diagonal inversa (\) se usa para los caracteres de escape, por lo que el motor de expresiones regulares no los analiza.

Los caracteres siguientes están reservados: []().\^$|?*+{}.

Deberá usar estos caracteres de escape en los patrones para que coincidan con ellos en las cadenas de entrada.

PowerShell
# This returns true and matches numbers with at least 2 digits of precision.
# The decimal point is escaped using the backslash.
'3.141' -match '3\.\d{2,}'

Hay un método estático de la clase regex que puede escapar texto por usted.

PowerShell
[regex]::escape('3.\d{2,}')
Output
3\.\\d\{2,}

Nota

Esto hace que se escapen todos los caracteres de expresión regular reservados, incluidas las barras diagonales inversas existentes que se usan en las clases de caracteres. Asegúrese de usarlo solo en la parte del patrón en la que necesita escape.

Otros escapes de caracteres

También hay escapes de caracteres reservados que puede usar para coincidir con tipos de caracteres especiales.

Estos son algunos escapes de caracteres usados habitualmente:

Escape de caracteres Descripción
\t Coincide con una pestaña
\n Coincide con una nueva línea
\r Coincide con un retorno de carro

Grupos, capturas y sustituciones

Las construcciones de agrupamiento separan una cadena de entrada en subcadenas que se pueden capturar u omitir. Las subcadenas agrupadas se denominan subexpresiones. De forma predeterminada, las subexpresiones se capturan en grupos numerados, aunque también puede asignarles nombres.

Una construcción de agrupación es una expresión regular entre paréntesis. Se captura cualquier texto que coincida con la expresión regular incluida. En el ejemplo siguiente se divide el texto de entrada en dos grupos de captura.

PowerShell
'The last logged on user was CONTOSO\jsmith' -match '(.+was )(.+)'
Output
True

Use la $Matches variable automática Hashtable para recuperar el texto capturado. El texto que representa toda la coincidencia se almacena en la clave 0. Es importante tener en cuenta que la $Matches tabla hash solo contiene la primera aparición de cualquier patrón de coincidencia.

PowerShell
$Matches.0
Output
The last logged on user was CONTOSO\jsmith

Las capturas se almacenan en claves numéricas de enteros que aumentan de izquierda a derecha. Capture 1 contiene todo el texto hasta el nombre de usuario, capture 2 contiene solo el nombre de usuario.

PowerShell
$Matches
Output
Name                           Value
----                           -----
2                              CONTOSO\jsmith
1                              The last logged on user was
0                              The last logged on user was CONTOSO\jsmith

Importante

La 0 clave es un entero. Puede usar cualquier método hashtable para tener acceso al valor almacenado.

PS> 'Good Dog' -match 'Dog'
True

PS> $Matches[0]
Dog

PS> $Matches.Item(0)
Dog

PS> $Matches.0
Dog

Capturas con nombre

De forma predeterminada, las capturas se almacenan en orden numérico ascendente, de izquierda a derecha. También puede asignar un nombre a un grupo de captura. Este nombre se convierte en una clave en la $Matches variable automática Hashtable .

Dentro de un grupo de captura, use para ?<keyname> almacenar los datos capturados bajo una clave con nombre.

PS> $string = 'The last logged on user was CONTOSO\jsmith'
PS> $string -match 'was (?<domain>.+)\\(?<user>.+)'
True

PS> $Matches

Name                           Value
----                           -----
domain                         CONTOSO
user                           jsmith
0                              was CONTOSO\jsmith

PS> $Matches.domain
CONTOSO

PS> $Matches.user
jsmith

En el ejemplo siguiente se almacena la entrada de registro más reciente en Seguridad de Windows Log. La expresión regular proporcionada extrae el nombre de usuario y el dominio del mensaje y los almacena en las claves:N para name y D para domain.

PowerShell
$log = (Get-WinEvent -LogName Security -MaxEvents 1).message
$r = '(?s).*Account Name:\s*(?<N>.*).*Account Domain:\s*(?<D>[A-Z,0-9]*)'
$log -match $r
Output
True
PowerShell
$Matches
Output
Name                           Value
----                           -----
D                              CONTOSO
N                              jsmith
0                              A process has exited....

Para obtener más información, vea Construcciones de agrupamiento en expresiones regulares.

Sustituciones en expresiones regulares

El uso de expresiones regulares con el -replace operador permite reemplazar dinámicamente texto mediante texto capturado.

<input> -replace <original>, <substitute>

  • <input>: cadena en la que se va a buscar
  • <original>: expresión regular que se usa para buscar en la cadena de entrada
  • <substitute>: expresión de sustitución de expresiones regulares para reemplazar las coincidencias encontradas en la cadena de entrada.

Nota

Los <original> operandos <substitute> y están sujetos a reglas del motor de expresiones regulares, como el escape de caracteres.

Se puede hacer referencia a grupos de captura en la <substitute> cadena. La sustitución se realiza mediante el carácter $ antes del identificador de grupo.

Dos maneras de hacer referencia a los grupos de captura son por número y por nombre.

  • Por número : los grupos de captura se numeran de izquierda a derecha.

    PowerShell
    'John D. Smith' -replace '(\w+) (\w+)\. (\w+)', '$1.$2.$3@contoso.com'
    Output
    John.D.Smith@contoso.com

  • Por nombre : también se puede hacer referencia a los grupos de captura por nombre.

    PowerShell
    'CONTOSO\Administrator' -replace '\w+\\(?<user>\w+)', 'FABRIKAM\${user}'
    Output
    FABRIKAM\Administrator

La $& expresión representa todo el texto coincidente.

PowerShell
'Gobble' -replace 'Gobble', '$& $&'
Output
Gobble Gobble

Advertencia

Puesto que $ el carácter se usa en la expansión de cadenas, deberá usar cadenas literales con sustitución o $ escape del carácter al usar comillas dobles.

PowerShell
'Hello World' -replace '(\w+) \w+', '$1 Universe'
"Hello World" -replace "(\w+) \w+", "`$1 Universe"
Output
Hello Universe
Hello Universe

Además, si desea tener como $ carácter literal, use $$ en lugar de los caracteres de escape normales. Al usar comillas dobles, sigue usando el escape de todas las instancias de para $ evitar una sustitución incorrecta.

PowerShell
'5.72' -replace '(.+)', '$$$1'
"5.72" -replace "(.+)", "`$`$`$1"
Output
$5.72
$5.72

Para obtener más información, vea Sustituciones en expresiones regulares.
